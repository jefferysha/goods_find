# 二手倒卖团队平台重构设计

> 日期：2026-02-13
> 状态：设计确认，待实施

## 项目定位

面向二手倒卖小团队的智能监控分析平台。核心价值：**自动发现低价好货，精确计算利润空间，高效协作完成交易**。

团队特征：
- 有一批长期监控的核心品类，同时临时追热点
- 通过新品价、二手行情价、目标出货价等多维度综合判断是否值得收
- 核心痛点：信息分散、基准价管理麻烦、缺少利润视角

---

## 一、页面结构与侧边栏

共 15 个页面，6 个分组，围绕 **发现 → 定价 → 交易 → 协作** 的工作流组织。

```
📊 总览
   └─ 仪表盘            经营驾驶舱，以利润为核心的全局数据

🔍 发现商品
   ├─ 捡漏雷达          按利润空间排序，最值得收的商品
   ├─ 溢价地图          按溢价率维度分析品类定价分布
   └─ 全部结果          现有结果页增强版，保留筛选/排序/导出

💰 价格管理
   ├─ 价格本            品类基准价 + 费用模板 + 自动算收购区间
   ├─ 行情走势          各品类二手价格趋势图
   └─ 竞品观察          同品类其他卖家的定价分布

📦 交易管理
   ├─ 采购清单          标记想收的商品，分配给成员
   ├─ 库存台账          已收货品的出入库记录
   └─ 利润核算          单品/批次的成本、售价、实际利润

👥 团队
   └─ 团队工作台        各成员待办、已收清单、业绩统计

⚙️ 系统
   ├─ 任务管理          监控任务配置
   ├─ 账号管理          平台账号管理
   ├─ 智能提醒          提醒规则
   ├─ 运行日志
   └─ 系统设置
```

## 二、自动定价模型（核心引擎）

每个品类在价格本里定义一套定价模板，系统自动为每个匹配商品计算收购区间。

### 2.1 定价模板结构

```
品类：MacBook Pro M2 14寸
├─ 基准价格
│   ├─ 新品参考价：        ¥14,999（京东自营价）
│   ├─ 二手行情价：        ¥9,500（市场均价，可自动从历史数据算）
│   └─ 目标出货价：        ¥10,000（计划卖出的价格）
│
├─ 费用模板
│   ├─ 运费：              ¥30（固定金额）
│   ├─ 整备/翻新费：       ¥100（固定金额）
│   ├─ 平台手续费：        5%（按出货价百分比）
│   └─ 其他费用：          ¥0
│
├─ 利润要求
│   ├─ 最低利润率：        15%
│   └─ 理想利润率：        25%
│
└─ 自动计算结果
    ├─ 总费用：            ¥30 + ¥100 + ¥500(手续费) = ¥630
    ├─ 收购上限：          ¥10,000 - ¥630 - 15%利润 = ¥7,870
    ├─ 理想收购价：        ¥10,000 - ¥630 - 25%利润 = ¥6,870
    └─ 收购区间：          ¥6,870 ~ ¥7,870
```

### 2.2 计算公式

```
总固定费用 = 运费 + 整备费 + 其他费用
平台手续费 = 目标出货价 × 手续费率
总费用 = 总固定费用 + 平台手续费

收购上限 = 目标出货价 - 总费用 - (目标出货价 × 最低利润率)
理想收购价 = 目标出货价 - 总费用 - (目标出货价 × 理想利润率)
收购区间 = [理想收购价, 收购上限]
```

### 2.3 使用方式

- **批量配置**：价格本页面是表格视图，每行一个品类，可直接在表格里编辑所有字段（类似 Excel）
- **单品微调**：在结果页/捡漏雷达里，点击某个商品可覆盖该品类的默认值（如成色差，整备费改为 ¥200）
- **自动匹配**：商品通过搜索关键词自动关联到对应品类的定价模板
- **行情自动更新**：二手行情价可选"手动设定"或"自动取最近 7 天中位价"

### 2.4 每个商品自动获得的标注

```
商品：MacBook Pro M2 14寸 95新  售价 ¥7,200
├─ 收购区间：¥6,870 ~ ¥7,870
├─ 判定：区间内（可收）
├─ 预估利润：¥10,000 - ¥7,200 - ¥630 = ¥2,170
├─ 预估利润率：30.1%
└─ 相对行情：比行情价低 24.2%
```

## 三、捡漏雷达

目标：打开就知道该行动，不用翻。

### 3.1 布局

**顶部汇总条：**
- 今日新增商品数、在收购区间内的数量、预估总利润

**主体：表格视图**

| 商品标题 | 平台 | 售价 | 收购区间 | 预估利润 | 利润率 | AI评分 | 状态 | 操作 |
|---------|------|------|---------|---------|-------|-------|------|------|
| MacBook Pro M2 95新 | 闲鱼 | ¥7,200 | ¥6.8k~7.8k | ¥2,170 | 30% | 推荐 | 可收 | [加入采购][忽略] |
| iPhone 15 全新未拆 | 转转 | ¥4,800 | ¥3.5k~4.2k | -¥250 | — | 偏贵 | 超出 | [忽略] |

### 3.2 行为

- 默认只显示"可收"的商品（售价在收购区间内），可切换显示全部
- 支持按利润率、利润金额、AI评分排序
- 每行可展开，查看商品图片、AI分析详情、卖家信息
- 右侧操作：一键加入采购清单、标记忽略、分配给成员

### 3.3 筛选条件

品类（关键词）、平台、利润率范围、价格范围、仅看AI推荐、时间范围

目标：回答"哪些品类现在溢价高，哪些有价格洼地"。

### 4.1 品类溢价概览（卡片网格）

```
┌─ MacBook Pro M2 ──────────┐
│ 在监控：23件                │

└────────────────────────────┘
```

### 4.2 溢价分布图（选中品类后展示）

- 横轴：价格，纵轴：商品数量
- 标注收购区间（绿色带）、行情价（蓝线）、新品价（灰线）
- 每个点可点击查看具体商品

### 4.3 跨品类对比表

选多个品类，横向对比平均溢价率、可收占比、利润空间，帮助决定精力往哪个品类投。

## 六、价格管理

### 6.1 价格本

表格视图，每行一个品类，列：

| 品类名称 | 关联关键词 | 新品参考价 | 二手行情价 | 行情来源 | 目标出货价 | 运费 | 整备费 | 手续费率 | 其他费用 | 最低利润率 | 理想利润率 | 收购区间 | 操作 |
|---------|----------|----------|----------|---------|----------|------|-------|---------|---------|----------|----------|---------|------|

- 所有字段可直接在表格中编辑（inline editing）
- 收购区间列为自动计算，实时更新
- 支持批量操作：选中多行 → 统一设置费用模板/利润率
- 支持导入/导出（CSV/Excel）
- 行情来源可选：手动 / 自动（最近7天中位价）

### 6.2 行情走势

- 品类选择器（单选或多选对比）
- 折线图：选中品类最近 30/60/90 天的二手均价、中位价走势
- 标注关键事件（如新品发布日，价格通常会跌）
- 数据来源：从历史爬取结果自动聚合

### 6.3 竞品观察

- 选中品类后，展示该品类所有在售商品的价格分布
- 按卖家分组，看各卖家的定价策略
- 价格区间分布直方图
- 数据来源：爬取结果中同品类的所有商品

## 七、交易管理

### 7.1 采购清单

在捡漏雷达/结果页点"加入采购"后，商品进入此页面。

**表格：**

| 商品标题 | 售价 | 预估利润 | 来源平台 | 负责人 | 状态 | 加入时间 | 操作 |
|---------|------|---------|---------|-------|------|---------|------|

**状态流转：** 新加入 → 待联系 → 议价中 → 已收货 → 放弃

- 点"已收货"弹出表单，填写实际收购价，自动转入库存
- 可按负责人筛选，每个人看自己的待办
- 支持批量分配：选中多个商品 → 分配给某成员
- 商品链接一键跳转到原平台

### 7.2 库存台账

商品从采购清单"已收货"后自动进入。

**表格：**

| 商品名称 | 收购价 | 总成本 | 挂牌价 | 状态 | 库龄 | 负责人 | 操作 |
|---------|-------|-------|-------|------|------|-------|------|

**状态流转：** 入库/整备中 → 在售 → 已出 / 退货

- 总成本 = 收购价 + 费用模板自动计算（可在此微调）
- 点"已出"时填写实际售价和售出渠道，自动进入利润核算
- 库龄预警：超过 N 天未售出的标红提醒，阈值可配置
- 顶部汇总：总库存件数、总成本、预估总货值

### 7.3 利润核算

所有"已出"的商品在此汇总。

**顶部数据卡片：**

| 本月已出 | 总营收 | 总成本 | 净利润 | 平均利润率 |
|---------|-------|-------|-------|----------|
| 32件 | ¥285,000 | ¥218,000 | ¥67,000 | 23.5% |

**明细表格：**

| 商品名称 | 收购价 | 费用 | 总成本 | 售价 | 净利润 | 利润率 | 售出日期 | 渠道 |
|---------|-------|------|-------|------|-------|-------|---------|------|

**分析维度：**
- 按时间范围（本周/本月/自定义）、品类、成员筛选
- 品类利润排行：哪个品类最赚钱、哪个亏钱
- 成员业绩：每个人收了多少、卖了多少、赚了多少
- 导出报表（CSV/Excel）

## 十、完整数据流

```
1. 价格本定义品类基准价和费用模板
2. 任务管理配置爬虫任务（关联品类关键词）
3. 爬虫定时抓取 → AI 多模态分析
4. 自动匹配品类 → 计算收购区间、预估利润
5. 捡漏雷达 / 溢价地图 / 全部结果 展示分析后的商品
6. 团队成员发现好货 → 加入采购清单 → 分配负责人
7. 联系卖家议价 → 确认收货（填实际收购价）→ 入库
8. 整备上架 → 售出（填实际售价）→ 利润核算
9. 仪表盘 / 团队工作台 展示全局经营数据
```

## 十一、对现有系统的改动总结

### 保留不变
- 任务管理、账号管理、智能提醒、运行日志、系统设置 — 功能保留，归入"系统"分组
- 爬虫核心逻辑（spider_v2.py、scraper.py）— 不变
- AI 分析服务 — 不变

### 需要改造
- **结果页**：增加溢价标注、利润显示、列表视图、加入采购操作
- **仪表盘**：从通用统计改为利润驱动的经营驾驶舱
- **收藏对比**：功能并入采购清单，不再作为独立页面
- **PricingService**：从手动设基准价改为基于定价模板自动计算

### 需要新建

**前端页面（7个新页面）：**
- 捡漏雷达
- 溢价地图
- 价格本
- 行情走势
- 竞品观察
- 采购清单
- 库存台账
- 利润核算
- 团队工作台

**后端服务：**
- `PriceBookService` — 定价模板 CRUD、收购区间计算
- `PurchaseService` — 采购清单管理、状态流转
- `InventoryService` — 库存出入库管理
- `ProfitService` — 利润核算、报表统计
- `TeamService` — 团队成员管理、任务分配、业绩统计

**后端 API 路由：**
- `/api/price-book/` — 定价模板管理
- `/api/purchases/` — 采购清单
- `/api/inventory/` — 库存管理
- `/api/profit/` — 利润核算
- `/api/team/` — 团队管理

**数据模型：**
- `PriceBook` — 品类定价模板
- `FeeTemplate` — 费用模板
- `PurchaseItem` — 采购清单项
- `InventoryItem` — 库存项
- `SaleRecord` — 销售记录
- `TeamMember` — 团队成员

**数据存储：**
- 扩展现有 SQLite（data/monitor.db），新增对应表
